# xlkit 設計骨子

## 方向性

### ターゲット・設計思想
| 項目 | 決定 |
|------|------|
| ターゲット | 自分用 + OSSとして本気で公開 |
| 設計思想 | バランス型（基本シンプル、必要なら拡張可能） |
| 目的 | データエクスポート + 帳票・レポート生成 |

### スコープ（v1.0）
| 機能 | 対応 |
|------|------|
| テーブル出力（ヘッダー + データ行 + スタイル + マージ） | ✅ |
| Excel読み込み・解析機能 | ✅ |
| 画像挿入 | ✅ |
| チャート | ❌ スコープ外 |

### Excel解析機能の用途
- **テスト検証用**: 生成したExcelが正しいかプログラムで検証
- **ライブラリ機能として提供**: ユーザーがExcelを読み込んでデータ抽出できる

### 技術スタック
| 項目 | 選定 | 理由 |
|------|------|------|
| 言語 | TypeScript | 型安全 |
| 依存 | ExcelJS | 実績あり |
| ビルド | tsup | ESM+CJS両対応、設定簡単 |
| リンター | Biome.js | 高速、設定シンプル |
| テスト | Vitest | 高速、Viteと相性良い |
| デモ | React + Vite | 型安全、エコシステム充実 |

### プロジェクト構成
- **モノレポ構成**: ライブラリ本体 + デモサイトを同一リポジトリで管理
- **ライブラリ名**: xlkit（維持）
- **公開先**: npm

### その他の検討事項
- モノレポ構成の詳細（packages/の分け方、pnpm workspace）
- ディレクトリ構成（src/内のモジュール設計）
